When you make anything with SnapEngine, you must first create an `Engine` instance. 
You can spawn multiple `Engine` instances on a single page, and for the most part they operate in isolation from each other.
<br/>

An engine will always need a container element assigned to it. This element defines the engine’s active area for input handling and rendering.

```typescript
import { Engine } from "snapengine";
const container = document.getElementById("engine-container");
const engine = new Engine();
engine.element = container;
```

You may have noticed the `element` property is set after instantiating the `Engine`. 
This is because SnapEngine is designed to integrate with frontend frameworks like React or Svelte,
many of which create DOM elements after the component logic runs.

```svelte
<script lang="ts">
   import { Engine } from "snapengine";
   let engine: Engine;
   engine = new Engine();
</script>
<div bind:this={engine.element} class="engine-container"></div>
```
```tsx
import React, { useRef, useEffect } from "react";
import { Engine } from "snapengine";

export function EngineComponent() {
   const containerRef = useRef<HTMLDivElement>(null);
   const engine = useRef<Engine | null>(null);
   useEffect(() => {
      engine.current = new Engine();
      if (containerRef.current) {
         engine.current.element = containerRef.current;
      }
   }, []);
   return <div ref={containerRef} className="engine-container"></div>;
}
```
```html
<div id="engine-container" class="engine-container"></div>
<script type="module">
   import { Engine } from "snapengine";

   const container = document.getElementById("engine-container");
   const engine = new Engine();
   if (container) {
      engine.element = container;
   }
</script>
```
> [!WARNING]
> Until `element` is set, the engine and any objects within it won’t be able to function properly.

## Global Manager

In addition, all `Engine` instances share a single global object called the `GlobalManager`. 
The `GlobalManager` is automatically created when the first `Engine` is instantiated, and every subsequent `Engine` instance will reference the same `GlobalManager` automatically.
This singleton manages the shared render loop, global input events, etc., which we will cover in more detail in later sections.
