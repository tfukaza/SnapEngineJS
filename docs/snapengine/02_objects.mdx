Every entity managed by an `Engine` inherits from `BaseObject`.
This class does not have a visual representation itself, but provides the foundational functionality for all objects in the engine.
<br/>

BaseObjects are added to an engine by instantiating them with a reference to the engine:

```typescript
import { Engine, BaseObject } from "snapengine";
const engine = new Engine();
const obj = new BaseObject(engine);
```

When an object is created, the global manager automatically assigns it a unique ID. This ID will be unique not only within the engine, but across all engines on the page.

Objects that have a DOM element assigned to them should use the `ElementObject` class instead:

```typescript
import { Engine, ElementObject } from "snapengine";
const engine = new Engine();
const elemObj = new ElementObject(engine);
elemObj.element = document.getElementById("my-element");
```

### Object Hierarchy
Every object can have a parent and multiple children, forming a hierarchical tree structure.
To set the parent of an object, pass the parent as the second constructor argument:

```typescript
const parentObj = new BaseObject(engine);
const childObj = new BaseObject(engine, parentObj);
```

### Transform Properties

Every object has built-in transform properties that specify its position in world coordinates.
```typescript
// Set the world position of the object:
obj.transform.x = 100;
obj.transform.y = 200;
// Or update multiple transform fields at once:
obj.worldTransform = { x: 100, y: 200 };
```
> [!NOTE]
> Properties for scaling and rotation are currently being considered, but may not be implemented in the final release.

The position is not in viewport coordinates, but rather in the world coordinate space of the engine.
Details on world coordinates will be explained in the ["Camera Control" section](./04_camera), but for now just know that the world coordinate space essentially means position on screen.
<br/>

### Coordinate System

SnapEngine automatically keeps world and local positions in sync. When you set an object's world position, its local position (relative to parent) is automatically computed, and vice versa.

```typescript
// Setting world position automatically updates local position
obj.worldTransform = { x: 150, y: 250 };
console.log(obj.transform.x, obj.transform.y); // Local transform, computed based on parent

// Setting local position automatically updates world position
obj.localTransform = { x: 50, y: 50 };
console.log(obj.worldTransform.x, obj.worldTransform.y); // Computed from parent.world + local
```

When a parent moves, all children's world positions cascade automatically while preserving their local offsets:

```typescript
const parent = new BaseObject(engine);
const child = new BaseObject(engine, parent);
child.localTransform = { x: 50, y: 50 };

parent.worldTransform = { x: 100, y: 100 };
console.log(child.worldTransform.x, child.worldTransform.y); // 150, 150 - automatically updated
console.log(child.transform.x, child.transform.y); // 50, 50 - preserved
```

If an object has no parent, its local and world positions are identical.

For `ElementObject` instances that need to read their position from the DOM, use `readDom()` and `saveDomProperety()` in a read stage:

```typescript
elemObj.schedule(() => {
  elemObj.readDom();
  elemObj.saveDomProperety("READ_1");
}, { stage: "READ_1" });
```

On the note of being different than typical game engines, **the transform properties of objects are not automatically applied to the actual DOM element of the object** - this must be done explicitly. We will cover how to do this in the ["Render Queues" section](./03_rendering).
